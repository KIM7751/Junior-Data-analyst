Day 5

[1-1-2] WAS 

        [1] 
            
            1) 클라이언트 URL을 통해 웹 브라우저로 인터넷에 요청 -> 2) 웹 서버에서 검색해 웹 문서를 확인했을때 정적데이터(HTML, JAVA Script) 직접 처리
        
            -> 3) 웹 브라우저 전송(해석)
            
        
        [2] 
        
            1) 클라이언트 URL을 통해 웹 브라우저로 인터넷에 요청 -> 2) 웹 서버에서 검색해 웹 문서를 확인했을때 동적데이터(ASP, JSP, PHP) 직접 처리 X
            
            -> 3) 웹 서버 자체 해석 또는 DB에 데이터를 요구 -> 4) DB 결과 전송 -> 5) 웹 서버 HTML 문서로 전환 -> 6) 웹 브라우저 결과 전송

           
           1 ~ 2 = 웹 서버
           
           3 ~ 4 = 웹 컨테이너
           
           웹 서버 + 웹 컨테이너 = WAS(Web Application Server)
           
           
               개념 - 동적인 웹 사이트, 애플리케이션, 서비스 개발을 지원하기 위해 설계된 소프트웨어로 데이터 접근 관리, 
                      세션(서버와 내가 연결을 유지하는 것)관리, 트랜잭션 관리 등을 위한 라이브러리(= 지원 부품, 함수)를 제공
                 
                    - HTTP 세션 처리를 위한 웹 서버 기능뿐만 아니라 필수적인 기업 업무 까지 JAVA, EJB 컴포넌트 기반으로 구현 가능
                    
                    HTTP: 홈페이지 지원 프로토콜(= 약속)
                    
                    TCP/IP: 인터넷 프로토콜(= 약속) 
                            - SMTP 이메일 전송
                            - HTP  파일 전송
                            - HTTP URL 접속
