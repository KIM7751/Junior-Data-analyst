[1-1-2] OS - 시스템 소프트웨어는 CPU, 메모리, 디스크등을 관리 -> 자원에 한계가 있음 -> 최대 효율을 내기 위해 관리가 필요한 것
           - 시스템 소프트웨어로 자원(resource)를 관리
           - 자원 H/W: CPU(=중앙처리장치,프로세서), 메모리, I/O, 주변장치 
                  S/W
                  
           - 사용자의 편의성이 극대화됨(명령어 입력 방식 -> 클릭)
           - 초기 ROM(Read Only Memory)에서 시작
           - 현재 RAM
               
               - ROM: 비휘발성 메모리, 읽기만 가능, 수정 불가능
               - RAM: 휘발성 메모리, 일기 수정 가능, 휘발성 메모리를 극복하기 위해 디스크(C:)에 보관
               ------------------------------------------------------------------------------------- 주 기억장치(사람. 기억력)
               
               - 디스크 ---------------------------------------------------------------------------- 보조 기억장치(사람. 메모)
               
               - OS의 거듭되는 발전에 크기가 ROM에 크기가 모자라자 현재 디스크에 저장 후 프로세스하여 RAM에 띄움
                 (= 부트, 부팅한다, Boot Strapping Loader)
                 
               - 실행 순서
               
               전원 -> ROM : POST -> ROM : BIOS -> ROM : Boot -> 디스크 : Boot Strapping Loader -> Process -> RAM : Booting
               
        운영체제의 종류 - Windows(Close Sys, 소스공개X): 개인용PC, 중소 규모 서버, 태블릿, 유지 및 관리 비용 측면에서 강점을 가짐
                        - Unix(Open Sys, 소스공개O): 1980년대 하나의 본체에 여러 컴퓨터를 연결해서 사용 -> 더미 단말기, Multi User
                                                     대용량 처리, 안정성이 요구되는 서버, NAS, WorkStation 등에서 사용
                                                     신뢰할 수 있는 대용량 처리를 위해서는 유닉스 기반 선호
                                                     
                        - Linux(Open Sys): 중대 규모 서버 사용, 일반적으로 리눅스 기반 시스템이 하드위어 및 소프트웨어 소요 비용이 가장 적게 소요
                        - IOS
                        - Android
       
        
        운영체제 분석 시 고려사항 - 신뢰도: 오랜 시간 시스템 운영시 OS 고유의 장애 발생 가능성 확인
                                  - 성능: 다수의 동시 사용자 요청 처리 가능 확인, 대량 파일 작업 처리 가능 확인
                                  - 기술 지원: 공급 벤터(유통업체)들의 안정적인 기술 지원 여부 확인
                                  - 주변 기기: 다수의 주변 기기 지원 여부 확인
                                  - 구축 비용: 지원 가능한 하드웨어 비용, 설치할 애플리케이션의 라이선스 정책 및 비용 확인
                              
        운영체제 현황 - 32bit(소용량, CPU에 32개bit 연결) 운영체제에서는 4GB 메모리까지 액세스 가능(사용자 메모리 2GB)
                      - 64bit(대용량, CPU에 64개bit 연결) 운영체제에서는 4GB 이상의 메모리까지 액세스 가능
                      
        
        CPU(중앙 처리 장치, 프로세서, 명령어 내장의 두 가지 방식이 있음)
            
            - CISC(Complex Instruction Set Computer): 소프트웨어적, 명령어가 무수히 많아질 수 있음
                                                      형식 변경 무수히 많음 -> 컴파일(해석)을 해야하므로 느리고 전력 소모가 많음
                                                      리눅스, 윈도우
                                                      
            - RISC(Reduced Instruction Set Computer): 하드웨어적, 명령어가 제한되어 있음
                                                      형식 변경 불가 -> 논리적이고 빠르며 전력 소모가 적음
                                                      유닉스
