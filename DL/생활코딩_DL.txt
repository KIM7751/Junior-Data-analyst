- 기계를 학습시켜서 인간의 판단 능력을 기계에게 위임하는 기술이 머신러닝 

- 머신러닝이라는 하나의 어휘 안에는 실로 많은 기술들이 포함되어 있음 

- 앞서 배웠던 ML에서는 크게 지도학습, 비지도학습, 강화학습이 있었음

- 우리가 텐서플로우를 이용해서 해결하려고 하는 문제는 지도학습 영역의 회귀문제, 분류문제임

- 회귀는 숫자로 된 결과를 예측하는 것이고, 분류는 범주형 즉 카테고리 형태의 결과를 예측하는 것 

- 위 두 문제를 푸는 방법들은 다양하며 이 방법들을 "머신러닝 알고리즘"이라고 함

- Decision Tree, Random Forest, KNN, SVM, Neural Network 같은 알고리즘이 있음

- 이 알고리즘들은 모두 머신러닝의 문제를 해결하는데 사용하는 알고리즘들임 

- 이 중 우리가 사용할 알고리즘은 "뉴럴 네트워크"임

- 뉴럴 네트워크는 사람의 두뇌가 동작하는 방법을 모방해서 기계가 학습을 할 수 있도록 고안된 알고리즘임

- 뉴런이라고 하는 세포들이 촘촘하게 연결되어 있고 연결되어 있는 신경망을 인공적으로 만들었다하여 "인공신경망"이라는 이름을 가지고 있음

- 지금은 인공신경망을 깊게 쌓아서 만들었다는 표현으로 "딥러닝"이라는 이름으로 많이 불리고 있음

- 딥러닝이 매우 어려운 문제들을 잘 해결할 수 있다는 사실을 발견하면서부터 딥러닝이라는 용어를 머신러닝을 대신하는 대표적인 용어로 사용하기 시작했고 

- 머신러닝도 인공지능을 대신하는 대표로써 사용되기 시작함

- 난해한 문제들을 오늘날 딥러닝을 이용해 풀 수 있게되었고 구체적인 딥러닝 원리를 몰라도 코드만 작성하면 딥러닝으로 문제를 해결할 수 있는 여러 도구들이 등장함

- 코딩의 세계에서는 이런 도구들을 "라이브러리"라고 부름

- 딥러닝 이론을 코딩으로 이용할 수 있도록 해주는 여러 라이브러리가 있음 

- Tensorflow, PyTorch, Caffe2, Theano 이 기술들은 모두 같은 목적으로 고안된 라이브러리임

- 이 중에서 우리는 Tensorflow를 이용해 딥러닝 이론으로 여러가지 문제를 해결할 것임 

- 수만 줄의 코드가 필요할 수 있는 일을 텐서플로우를 이용하면 몇 줄의 코드로 처리할 수 있게됨

- 이리저리 만져보고 흔들어보고 던져보고 필요한 만큼 경험하면서 비로소 잘 사용할 수 있게 되는 것임

- 굉장히 복잡한 도구인 스마트폰을 직접적인 경험을 통해 사용법을 배워나가는 것임

- 그 과정은 고통스럽지 않고 오히려 즐거움 

- 딥러닝도 이렇게 접근해보자 

- 원인이 되는 간단한 코드를 작성하며 경험하고

- 결과로써 코드의 동작과 학습과정을 구경함

- 그리고 해당 코드를 어떻게 이용하면 좋을지 추측해봄

- 이 과정을 반복하며 딥러닝을 구현하는 시간을 가질 것임

- 그 과정에서 코드와 알고리즘의 동작에 익숙해지는 것

지도학습의 빅피쳐

- 지도학습을 하기위해서는 "과거의 데이터"가 필요함

- 과거의 데이터 속에서 "원인"과 "결과"를 인식하는 것

- 데이터 과학과 머신러닝의 분야에서는 이런 원인을 "독립변수", 결과를 "종속변수"라고 부름

- "모델의 구조"를 만듦, 독립변수도 하나이고 종속변수도 하나인 데이터를 가지고 있을때 입력과 출력이 각각 하나인 모델의 구조가 됨

- 데이터를 준비하고 모델의 모양구성을 마쳤다면 "모델을 데이터에 딱 맞도록 함"

- 이런 과정을 "Fitting한다"고 하고 옷이 잘 어울릴 때 Fit이 좋다고도 함 = 학습

- 모델을 완성하고 이용함

- 예측하고 싶은 데이터 독립변수를 모델에 넣으면 종속변수가 결과로 나오게 됨
